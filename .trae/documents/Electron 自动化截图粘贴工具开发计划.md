这是一个非常好的自动化需求。使用 Electron 开发是一个合适的选择，因为它可以方便地进行跨平台分发，并且拥有丰富的 Node.js 生态来处理系统底层操作（截图、模拟键鼠）。

针对你的需求，我建议采用 **“内置截图 + 模拟操作”** 的全自动化方案，而不是调用微信截图。因为调用外部截图工具（如微信）很难实现全自动化的“选区-确认”流程，容易中断。

### 核心技术方案

1.  **框架**：Electron + React (UI) + Vite (构建工具)。
2.  **截图实现**：使用 Electron 自带的 `desktopCapturer` 配合 `sharp` 或 `jimp` 进行图片裁剪，或者直接使用 `screenshot-desktop` 库，实现无损截图。
3.  **模拟操作**：使用 `@nut-tree/nut-js` 或 `robotjs` 来模拟鼠标点击、键盘按键（Ctrl+V, PageDown, Alt+Tab 等）。
4.  **剪贴板**：使用 Electron 的 `clipboard` API 直接写入图片数据，保证不压缩质量。
5.  **交互设计**：
    *   提供一个“配置模式”，让用户框选“右边幻灯片区域”和点击“左边粘贴位置”。
    *   一键启动/停止（支持全局快捷键，如 F10 停止）。

### 实施计划

我将分四个阶段来实现这个工具：

#### 阶段一：项目初始化与环境搭建
1.  在当前目录初始化 Electron + Vite + React 项目结构。
2.  配置 `electron-builder` 以便后续打包成可执行文件 (.exe)。
3.  安装核心依赖：`@nut-tree/nut-js` (模拟键鼠), `jimp` (图片处理), `uiohook-napi` (可选，用于监听快捷键)。

#### 阶段二：UI 界面开发
1.  开发主控制面板：
    *   设置“循环次数”或“无限循环”。
    *   设置“操作间隔时间”（毫秒），防止操作过快软件响应不过来。
    *   “开始任务”和“停止任务”按钮。
    *   状态日志显示。
2.  开发“位置校准”向导：
    *   功能 A：**拾取右侧区域**。点击按钮后，用户在屏幕上框选右侧幻灯片的显示区域。
    *   功能 B：**拾取左侧焦点**。用户点击左侧窗口的任意位置，记录坐标（用于粘贴前激活窗口）。

#### 阶段三：核心自动化逻辑实现 (Main Process)
1.  实现 `screenshot` 函数：根据用户框选的区域截取屏幕图像。
2.  实现 `automation` 流程控制：
    *   **Step 1**: 截取右侧区域图片 -> 写入系统剪贴板。
    *   **Step 2**: 移动鼠标到左侧坐标 -> 点击激活窗口 -> 模拟按下 `Ctrl + V`。
    *   **Step 3**: 模拟按下左侧窗口的翻页键（如 `PageDown` 或 `Enter`）。
    *   **Step 4**: 移动鼠标回右侧区域中心 -> 点击激活窗口 -> 模拟按下右侧窗口的翻页键。
    *   **Step 5**: 等待间隔，循环执行。
3.  注册全局快捷键（如 `Alt+Q` 强制停止），防止程序失控。

#### 阶段四：验证与打包
1.  在本地进行模拟测试（使用记事本或画图软件模拟）。
2.  修复可能出现的权限问题（Windows 上模拟输入可能需要管理员权限）。
3.  打包生成 `.exe` 安装包或绿色版文件。

这个方案可以完全满足“不压缩图片质量”、“自动循环”、“别人下载也能用”的需求。
